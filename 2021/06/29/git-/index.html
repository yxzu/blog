git

序：

随着面临多人协作，团队项目分支管理，在踩过一个不懂git命令的坑以后，终于发现对于代码管理工具git， 不能停留在commit、pull、push…阶段，来梳理一波

了解一下基本

git优势（分布式代码管理仓库）：

  • 本地提交，也不需要依赖网络，并且会将每次提交自动备份到本地。每个开发者都可以从远程仓库clone一份到本地，并会把提交历史一并拿过来。

git文件的状态：

  • 在Git中文件大概分为三种状态：已修改（modified）、已暂存（staged）、已提交（committed）
    • 修改：当工作目录中的文件被修改，则把修改的文件加入到modified区域
    • 暂存：使用add命令将工作目录中修改的文件提交到暂存区，等候被commit
    • 提交：将暂存区文件commit至Git目录中永久保存

commit：

  • 在Git中每一次commit都会生成一个节点, 而每个节点都会有一个哈希值作为唯一标识， 多次commit形成一个节点链

HEAD：

  • HEAD是Git中是非常重要的，可以将它称为指针，它可以指向任意一个节点，并且指向的节点始终为当前工作目录

远程仓库：

  • Git会把代码以及history保存在本地，但最终还是要提交到远程仓库。可以把远端代码clone到本地，同时也会将提交历史、分支、HEAD等状态一并同步到本地，但这些状态并不会实时更新

命令整理

git文件的状态：

git add xxx  (添加'xxx'文件到暂存区)
git add . (添加所有文件到暂存区)
git checkout -- xxx (撤销xxx工作区改动)
git reset HEAD xxx (清空暂存区)
git commit -m "description" （提交并生成一个新的提交节点）
git branch xxx (创建一个名称为xxx新的分支)
git checkout xxx  (切换到名称为xxx的分支)
git checkout -b xxx (创建并切换至xxx分支)
git branch -d xxx (删除xxx分支)
git merge xxx/hash (将某个分支或者节点合并至当前的分支)
git rebase xxx/hash （将某个分支或者节点合并至当前的分支）
git cherry-pick hash (选择某几个节点合并)
git reset HEAD~N (回退N个提交)
git clone address （远端复制代码）
git fetch address /xxx (下载至本地) 
git pull xxx (拉取代码)
git push xxx （推送至远端xxx分支）
git fetch address /xxx (下载至本地)
